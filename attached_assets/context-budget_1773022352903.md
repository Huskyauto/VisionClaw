# Context Budget Guide

## Token Allocation Strategy

Your context window is finite. Every token spent on workspace files is a token not available for conversation or tool results. Here's how to allocate.

### Recommended Split
| Category | Budget | Purpose |
|----------|--------|---------|
| System prompt | 10-15% | Instructions, persona, config |
| Workspace files | 15-20% | MEMORY.md, session-state.md, essential docs |
| Conversation | 40-50% | User messages + agent responses |
| Tool results | 20-25% | Search results, file contents, API responses |
| Buffer | 5-10% | Headroom before compaction triggers |

### Common Waste Patterns

**1. Loading everything at boot**
Problem: 5+ workspace files loaded every session, most never referenced.
Fix: Only auto-load IDENTITY.md, SOUL.md, and session-state.md. Load others on demand.
Savings: 30-50% of workspace budget

**2. Full file reads when you need 10 lines**
Problem: Reading entire files (2000+ lines) when you need one section.
Fix: Use offset/limit parameters. Read headers first, then target sections.
Savings: 80-90% per file read

**3. Verbose tool output**
Problem: Search results return full page content when you need titles + URLs.
Fix: Use compact output formats. Extract what you need, discard the rest.
Savings: 50-70% per tool call

**4. Conversation bloat**
Problem: Re-explaining context that should be in a file.
Fix: Write context to session-state.md once. Reference it instead of repeating.
Savings: Variable, often 20-30% of conversation budget

**5. Redundant compactions**
Problem: Compaction triggers, but the summary is nearly as large as the original.
Fix: Keep conversation focused. Long outputs go to files, not inline.
Savings: Prevents compaction cycles entirely

### Per-Model Context Windows

| Model | Context Window | Practical Limit (before compaction) |
|-------|---------------|-------------------------------------|
| Claude Opus 4 | 200K tokens | ~160K (80%) |
| Claude Sonnet 4 | 200K tokens | ~160K (80%) |
| Gemini 2.5 Flash | 1M tokens | ~800K (80%) |
| GPT-4o | 128K tokens | ~100K (80%) |

### Monitoring Commands

Tell your agent:
- "How much context am I using?" -- estimates current usage
- "What's eating my context?" -- identifies largest consumers
- "Optimize my context" -- suggests files to remove or shrink
- "How many compactions today?" -- tracks compaction frequency
